## Summary

`v0.1.15` published the cleaner official-source review loop for install-only
users. It does not add reviewed ProviderEvents and does not move or recreate
the signed `data-2026.06.11` immutable data tag.

This release includes reviewed-duplicate candidate suppression, source-state
only refresh routing, latest-entry developer-surface status filtering, and the
bounded official source-state refresh from PRs #144 through #147. It does not
change event-promotion authority, source-owner authority, release-token
boundaries, or the read-only MCP default.

```bash

uv pip install --python /tmp/apw-pypi-smoke-015/bin/python --refresh "ai-provider-watch==0.1.15"
cd /tmp/apw-pypi-smoke-015-run
/tmp/apw-pypi-smoke-015/bin/python - <<'PY'
from ai_provider_watch import __version__

assert __version__ == "0.1.15", __version__
assert freshness["package_version"] == "0.1.15", freshness
assert freshness["event_count"] == 45, freshness

PY
/tmp/apw-pypi-smoke-015/bin/apw freshness --summary
/tmp/apw-pypi-smoke-015/bin/apw operations launch-gate --package-version 0.1.15 --summary
/tmp/apw-pypi-smoke-015/bin/apw latest --limit 1
```

## Known Caveats

- The PyPI exact-version JSON exposed `0.1.15` before the resolver-visible
  Simple API response included the new files. The normal resolver install by
  exact version passed after refreshing the resolver cache.
- `v0.1.15` is a read-only package snapshot for source-review behavior and
  bundled data. It does not permit unattended event publication and does not
  change the policy that generated candidates require source-owner review before
  promotion.
