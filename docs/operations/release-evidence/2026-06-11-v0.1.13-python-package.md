## Summary

`v0.1.13` published the safe issue-triage and source-refresh gating CLI
surfaces for install-only users. The bundled reviewed feed remains the signed
`data-2026.06.11` snapshot with forty reviewed ProviderEvents and latest event
date `2026-06-11`.

This release adds `apw source review-needed` so scheduled source refreshes can
skip no-op candidate-review PRs while still uploading sanitized observation
artifacts. It also includes `apw event issue-triage` from PR #137 for safe
Missing provider event issue triage. It does not change event-promotion
authority, source-owner authority, release-token boundaries, or the read-only
MCP default.

## Known Caveats

- The PyPI exact-version JSON exposed `0.1.13` before the HTML Simple API
  response used by the resolver included the new files. The normal resolver
  install by exact version passed after the Simple API cache refreshed.
- `v0.1.13` is a read-only package snapshot for `data-2026.06.11`. It does not
  permit unattended event publication and does not change the policy that
  generated candidates require source-owner review before promotion.
