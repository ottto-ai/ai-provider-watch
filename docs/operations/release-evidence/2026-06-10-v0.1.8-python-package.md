## Known Caveats

- The first immediate `pip install ai-provider-watch==0.1.8` attempt saw the
  PyPI simple index before it had propagated and listed only `0.1.7`; a retry
  with `--no-cache-dir` succeeded after the simple index exposed `0.1.8`.
- The package does not expose `apw --version`; installed version was verified
  through Python import.
- `apw operations report --summary` reported `overall_status: fail` because the
  current reviewed source-state freshness and missing-source counts are below
  the operational target. This did not block the package patch because
  `v0.1.8` is a CLI workflow release, not a new data freshness declaration.
