## Summary

`v0.1.16` publishes the installable package snapshot for signed data release
`data-2026.06.11.1`. It bundles 56 reviewed ProviderEvents so install-only
users get the same OpenAI API changelog, Codex changelog, and OpenAI News event
coverage as same-day remote-feed consumers.

This release does not change event-promotion authority, source-owner authority,
release-token boundaries, or the read-only MCP default.

## Install Smoke

Fresh install smoke used Python 3.12.11 in `/tmp/apw-pypi-smoke-016`.

Commands that passed after installing `ai-provider-watch==0.1.16` from PyPI:

```bash
uv venv /tmp/apw-pypi-smoke-016 --python 3.12 --seed
uv pip install --python /tmp/apw-pypi-smoke-016/bin/python --refresh --index-url https://pypi.org/simple "ai-provider-watch==0.1.16"
cd /tmp
/tmp/apw-pypi-smoke-016/bin/python - <<'PY'
from ai_provider_watch import __version__, api
freshness = api.load_json_feed("freshness")
latest = api.load_events(limit=1)
assert __version__ == "0.1.16", __version__
assert freshness["package_version"] == "0.1.16", freshness
assert freshness["event_count"] == 56, freshness
print({"version": __version__, "event_count": freshness["event_count"], "latest_event_date": freshness["latest_event_date"], "latest_id": latest[0]["id"]})
PY
/tmp/apw-pypi-smoke-016/bin/apw validate
/tmp/apw-pypi-smoke-016/bin/apw index --check
/tmp/apw-pypi-smoke-016/bin/apw freshness --summary
/tmp/apw-pypi-smoke-016/bin/apw operations launch-gate --package-version 0.1.16 --summary
/tmp/apw-pypi-smoke-016/bin/apw latest --limit 1
```

## Known Caveats

- The PyPI exact-version JSON exposed `0.1.16` before the first resolver
  install attempt found it. The normal exact-version install passed after the
  Simple API included the files.
- `v0.1.16` is a read-only package snapshot for bundled data. It does not
  permit unattended event publication and does not change the policy that
  generated candidates require source-owner review before promotion.
