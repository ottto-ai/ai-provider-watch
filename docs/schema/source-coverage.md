# Source Coverage Schema

`SourceCoverageReport` is the public feed-health summary for APW source
coverage, source-state freshness, known parser gaps, and review backlog.

Use [Source Catalog](source-catalog.md) when you need the provider/source
support matrix and per-source validation details. Use source coverage when you
need a compact health report and warning list for release review.

The canonical artifact is:

```text
data/feeds/coverage.json
```

It is generated by `apw index`, bundled in Python package data, included in
release dry-run artifacts, and listed in `data/releases/*/checksums.txt`.

## Purpose

Consumers use source coverage metadata to answer:

- how many provider sources are registered and enabled;
- which enabled deterministic sources have source-state fingerprints;
- which official sources are blocked pending parser graduation;
- which sources are manual-review-only;
- how many reviewed events exist per provider;
- how much review-candidate backlog exists;
- which source coverage warnings should be considered before a data release.

## Contract

- `schema_version`: `apw.source_coverage.v0`.
- `summary`: counts for providers, sources, fetched enabled sources, blocked
  parser sources, reviewed events, candidate backlog, and warnings.
- `source_state`: path and latest retrieval metadata for
  `data/source-state/fingerprints.json`.
- `candidate_backlog`: review-candidate count by review status.
- `providers`: per-provider source, event, and backlog counts.
- `sources`: per-source coverage status, parser fixture count, source-state
  presence, and graduation blockers.
- `warnings`: machine-readable visibility warnings. Warnings do not grant
  publication authority and do not publish events.
- `coverage_policy`: explicit statement that the artifact contains no raw
  provider content.

## CLI

Print JSON:

```bash
apw source coverage
```

Print a compact human summary:

```bash
apw source coverage --summary
```

Validate that coverage metadata is present, schema-valid, and current:

```bash
apw validate
apw index --check
```
