# Feed Freshness Schema

`FeedFreshness` is the public provenance summary for generated APW feeds.

The canonical artifact is:

```text
data/feeds/freshness.json
```

It is generated by `apw index`, bundled in the Python package data, included in
release dry-run artifacts, and listed in `data/releases/*/checksums.txt`.

## Purpose

Consumers use freshness metadata to answer:

- which APW package version generated this feed;
- which data release or development release the feed belongs to;
- how many reviewed events are present;
- what the latest reviewed event date and observed timestamp are;
- when official-source fingerprint state was last retrieved;
- where to find the release manifest and checksum manifest;
- which feed and index artifacts can be hash-checked.

## Contract

Required fields:

- `schema_version`: `apw.feed_freshness.v0`.
- `generated_at`: deterministic generation timestamp for the feed set.
- `generated_by`: generator name and version.
- `package_version`: APW package version.
- `release_id`: `dev` or a CalVer data release ID such as
  `data-2026.06.05`.
- `data_tag`: matching data tag for CalVer releases, or `null` for `dev`.
- `event_count`: count of reviewed `ProviderEvent` records.
- `latest_event_date`: latest reviewed event date, or `null` for an empty feed.
- `latest_observed_at`: latest reviewed event observation timestamp.
- `source_state`: path, SHA-256, byte count, source count, and latest retrieval
  timestamp for `data/source-state/fingerprints.json`.
- `release_artifacts`: paths to the release manifest and checksum manifest.
- `feed_artifacts`: path, media type, SHA-256, and byte count for generated
  feed and index artifacts.
- `freshness_policy`: explicit statement that the artifact contains no raw
  provider content.

`FeedFreshness` stores hashes, counts, paths, and timestamps only. It must not
store raw provider HTML, source-page prose, authenticated console content,
screenshots, cookies, credentials, customer telemetry, or generated candidate
text.

## CLI

Print the machine-readable artifact:

```bash
apw freshness
```

Print the compact operator summary:

```bash
apw freshness --summary
```

Validate that freshness metadata is present, schema-valid, and current:

```bash
apw validate
apw index --check
```
