# JSON Feed 1.1

`data/feeds/feed.json` is the reviewed-events JSON Feed 1.1 artifact.

It is generated by `apw index`, bundled in the Python package data, included in
release dry-run artifacts, and listed in `data/releases/*/checksums.txt`.

## Purpose

Use this feed when a downstream reader, static site, agent dashboard, or feed
client wants a simple syndication format instead of the full APW
`ProviderEvent` array.

The feed contains reviewed APW events only. It does not contain raw provider
page text, generated candidate text, issue bodies, PR comments, credentials,
screenshots, customer telemetry, or authenticated console content.

## Contract

- `version`: `https://jsonfeed.org/version/1.1`.
- `items`: the 50 most recent reviewed `ProviderEvent` records.
- `items[].id`: stable APW event ID.
- `items[].url`: raw GitHub URL for the canonical event JSON file.
- `items[].external_url`: first official evidence URL for the event.
- `items[].content_text`: APW summary plus compact APW classification metadata.
- `items[].tags`: APW event kind, severity, providers, and reviewed event tags.
- `items[]._apw`: APW extension with event kind, provider refs, source
  authority, evidence hashes, compact impacts, and limitations.

The `_apw` extension intentionally repeats APW metadata so feed clients can
route items without fetching the full event JSON first.

## Validation

```bash
apw validate
apw index --check
```

`apw validate` verifies that `data/feeds/feed.json` is schema-valid and current.
